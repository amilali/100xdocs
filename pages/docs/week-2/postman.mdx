## What is Postman?

Postman is a powerful collaboration platform for API development. It simplifies the process of developing APIs and streamlines the testing of APIs by providing a user-friendly interface. Postman supports various HTTP methods, making it an ideal tool for testing and exploring RESTful APIs.

## How to Use Postman

1. **Download and Install:**
   - Visit the [Postman website](https://www.postman.com/) to download the application.
   - Choose the appropriate version for your operating system (Windows, macOS, or Linux).
   - Follow the installation instructions provided on the website.

2. **Create an Account (Optional):**
   - While it's not necessary to create an account, having one allows you to sync your Postman data across devices.
   - You can sign up for a free account on the Postman website.

3. **Launch Postman:**
   - After installation, open the Postman application on your computer.

4. **Interface Overview:**
   - Postman has a user-friendly interface with different sections for requests, collections, environments, etc.
   - The main areas are the request builder, where you can specify the request type, URL, headers, and body; and the response viewer, which displays the results.

5. **Create Requests:**
   - To test an API, create a new request by clicking on the "New" button.
   - Choose the request type (GET, POST, PUT, DELETE, etc.).


- **GET Request:**
   - Open Postman.
   - Set the request type to `GET`.
   - Enter the URL: `http://localhost:3000/`.
   - Send the request and observe the response.

- **POST Request:**
   - Set the request type to `POST`.
   - Enter the URL: `http://localhost:3000/`.
   - In the request body, add a JSON object with the property `isHealthy` to specify the kidney health.
   - Send the request and observe the response.

- **PUT Request:**
   - Set the request type to `PUT`.
   - Enter the URL: `http://localhost:3000/`.
   - Send the request and observe the response.

- **DELETE Request:**
   - Set the request type to `DELETE`.
   - Enter the URL: `http://localhost:3000/`.
   - Send the request and observe the response.

6. **Add Headers and Body:**
   - Customize your request by adding headers (e.g., Content-Type) and, if applicable, a request body.
   - For POST and PUT requests, you can use the body section to send data.
